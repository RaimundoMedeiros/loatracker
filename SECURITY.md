# Security Policy

## Supported Versions

Security fixes are provided for the latest code on the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead:

1. Open a private report via GitHub Security Advisories (preferred), or
2. Open an issue with minimal details asking for a private contact channel.

When reporting, include:

- Steps to reproduce
- Impact and affected area
- Possible mitigation (if known)

We will acknowledge reports as soon as possible and keep you updated during triage and remediation.
